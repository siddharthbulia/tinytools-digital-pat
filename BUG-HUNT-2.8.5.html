① Backend — the AI-quota cluster (highest leverage)

All four live in the same two files (supabase/schema.sql, functions/generate-image/index.ts) and shipped together as one migration + redeploy.

IDSeverityBugFix shipped
brls-1P2 Cross-user quota DoS. consume_generation is SECURITY DEFINER but had no explicit grant, so it kept Postgres's default EXECUTE-to-PUBLIC. It takes a caller-supplied p_uid and never checks auth.uid(), so any anon client could POST rpc('consume_generation',{p_uid:<victim>}) and drain a friend's daily image-generation cap. REVOKE EXECUTE … FROM public, anon, authenticated + GRANT … TO service_role. (Note: the suggested current_user='service_role' body-check is wrong for DEFINER functions — inside them current_user is the owner — so the grant is the real control.) Verified: anon now gets 401.
brls-2P2 Cap bypass via TOCTOU. The check was an unlocked SELECT then a separate increment — N parallel requests all read the same pre-increment count, all pass, all increment, blowing past the cap. Collapsed into one atomic, row-locked INSERT … ON CONFLICT … RETURNING that reserves and returns the post-increment count; an over-cap reservation is refunded.
SC-3P2 Failed generations silently burned quota. The counter was incremented before the OpenAI call and never refunded — a few transient 502s during one character (≈11–17 calls each) could eat a big slice of the 60/day cap. Also failed open when platform env was missing. Reserve-then-refund: the edge function reserves before the call and calls a new refund_generation RPC on every failure path; missing-env now fails closed. Redeployed (verify-jwt on).
brls-6P3 A friend could vanish. my_friends() INNER-joined profiles; a friendship whose counterpart profile row was transiently missing returned no row, so the client tore down that friend's presence channel. LEFT JOIN + coalesce defaults so a friendship always yields a friend; the live presence frame backfills the real name/character.

② Chipkoo & presence (the multiplayer core)

IDSeverityBugFix shipped
CHIPKOO-1P2 Unreleasable clinging pet. Re-picking the already-active Chipkoo menu item bumped clingEpoch but the cursorMode didSet didn't fire (same value), so the new epoch was never broadcast. A friend's pat (old epoch) could then never release the pet for the rest of the session. Made enterChipkoo() idempotent (guard cursorMode != .chipkoo) and disabled the already-active cursor menu item.
PS-1P2 Reconnect race leaks presence callbacks. Four reconnect triggers (sleep-wake, network, silent socket, manual) fired resubscribeAll() unserialized; on a lid-open/Wi-Fi flap two runs interleaved at every await, overwriting a live channel handle and double-firing publish(). Reentrancy guard (single run at a time, coalesced re-run) + gated all triggers on a real started flag set only after the initial subscribe completes.
PS-5P3 Clinging pet lingered after quit. The 45s heartbeat wasn't stopped on the quit path, so an in-flight tick could re-track() presence after untrackAll() — friends saw you online (and clinging) for up to 120s. New prepareForQuit() stops the heartbeat first + latches a quitting flag that no-ops any further track.
CHIPKOO-2P3 Stale clear survived a restart. A viewer's pending clearReq wasn't dropped when the owner left; since clingEpoch resets to 0 on relaunch, the stale request could release a brand-new cling that landed on the same epoch value. Null edgeClearReq.byFriend[owner] in the presence-leave handler.

③ Characters & sprites

IDSeverityBugFix shipped
CS-1P2 Stale art after regenerating a character. The sprite cache was write-once and never invalidated; clearCache() was dead code. Regenerating an existing character kept drawing the old sprites until relaunch. Added Sprites.clear(characterId:), called after generation; finishAddCharacter now force-re-renders when the active character is regenerated.
CS-2P2 Wedged to cat fallback. A failed neutral.png write was swallowed by try?; the character was still selected & persisted, but rendered the bundled cat — across relaunches, with no recovery. Stop swallowing the load-bearing write; validate the character dir exists before returning/persisting the id.
CS-3P3 Repeated disk probes. Missing optional frames (per-mood blinks) were never negatively cached, so every blink tick re-ran FileManager + a failing image load forever. Cache misses too ([String: NSImage?]) so each absent frame is probed at most once.

④ Roaming & mood

IDSeverityBugFix shipped
MR-2P2 Pet stranded at a 22px sliver. An off-screen "peek" persisted the sliver origin; if roaming was toggled off mid-peek the pet was left at the edge and relaunched there. saveOrigin now refuses off-screen positions (center-on-screen), setCalm rescues a stranded pet to the nearest corner, and visibleSomewhere is center-based so a saved sliver re-anchors.
MR-3P2 Pet freezes near a browser. A synchronous AppleScript ran on the main thread every 5s (and on every app switch) to read the active tab — a slow/unresponsive browser froze all pet motion, and the first call popped a blocking permission prompt. Cache the tab host per browser and only re-query on an explicit app switch or every ~18s instead of every tick.
MR-5P3 Double-pat cut the purr short. Each pat scheduled an un-cancelled timer; a second pat within 1.3s let the first timer end the purr early. Tag the purr with the pat counter; only the latest pat's timer ends it.
MR-4P3 Orphaned animation timer per removed friend. Unfriending didn't stop the friend pet's Animator timer (latent leak; would be a 60Hz leak for any future looping clip). FriendPet.close() stops the animator + added Animator deinit.
MR-1P3 Crash on a tiny display. wander() sampled CGFloat.random(in:) on a range that reverses on a very small/headless screen → fatal trap. Guard the bounds before sampling; skip the tick if the screen is too small.

⑤ UI & gestures

IDSeverityBugFix shipped
appkit-ui-5P3 Flick-taps swallowed. A tap with >5px of trackpad drift was classified as a drag — no purr/hearts, and a Chipkoo cling couldn't be released. Added a press-duration gate: a brief, low-travel contact registers as a pat (threshold raised to 10px).
appkit-ui-6P3 Rapid patting wiped new hearts. Each pat scheduled an un-cancelled hearts-clear; an earlier clear emptied hearts a later pat just added. Generation-token the clear so only the latest pat's timer clears.
appkit-ui-7P3 Fragile relaunch after Reset. Relaunch built a shell string with the bundle path double-quoted; a path with a quote/backslash would break it, leaving no running pet. Launch /usr/bin/open -n directly via Process with an argument array — no shell.
appkit-ui-8P3 Silent onboarding dead-end. If anonymous sign-in failed (offline first run), "Let's go" just re-enabled with zero feedback. start() now returns success; onboarding shows a "couldn't connect" error.

Fix order shipped

  1. brls-1 — revoke the quota RPC from clients (security; prerequisite for trusting the cap).
  2. brls-2 — make the cap atomic (same migration).
  3. SC-3 — reserve-then-refund so failures don't burn quota; fail closed.
  4. CHIPKOO-1 — idempotent enterChipkoo (user-visible multiplayer breakage).
  5. CS-1 / CS-2 — sprite-cache eviction + don't persist a wedged character.
  6. PS-1 — serialize the reconnect paths.
  7. MR-3 / MR-2 — kill the browser-query freeze; stop the stranded-pet.
  8. PS-5 / CHIPKOO-2 / brls-6 — presence-on-quit, stale clear, vanishing friend.
  9. Remaining P3 polish — gestures, hearts, purr, relaunch, onboarding, timers, wander guard, negative cache.
